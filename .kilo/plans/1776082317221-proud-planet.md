# Toplantı Notları Organizasyon Planı

## Proje Amacı
Toplantı notları ve WhatsApp yazışmalarını renkli, hiyerarşik ve tarihli bir yapıda organize etmek.

## Veri Kaynakları
- Toplantı Notları: https://dashboard.corteqs.net/toplanti
- WhatsApp İçerikleri: https://dashboard.corteqs.net/bawachat
- MVP Hedefleri: https://dashboard.corteqs.net/mvpitems (ilk liste akordeon formatında)
- MVP Genel Özet: https://dashboard.corteqs.net/mvp (içeriği mvpitems sayfasına "GENEL ÖZET" kartı olarak eklenecek, bu section silinecek)

## Görünüm Tasarımı (Örnekten Alınan)

### Ana Bölümler
- **Toplantı Özeti**: Renkli noktalarla kategorize edilmiş ana konular
- **Rezervasyon ve Event Yönetimi**: Ana bölüm başlıkları
- **Alt Bölümler**: WA güncellemeleri, dinamik form kısıtlamaları gibi

### Yapı Detayları
1. **Renk Kodlama** (Google Renkleri):
   - Kırmızı: Rezervasyon Sistemi
   - Yeşil: Kullanıcı Kısıtlamaları  
   - Mavi: Audit Kayıtları
   - Sarı: Veritabanı Tasarımı
   - Mor: MVP Hedefleri
   - Turkuaz: Reklam Modeli

2. **Tarih Etiketleme**:
   - "13 Nisan WA": WhatsApp güncellemeleri
   - "12 Nisan": Toplantı 2 maddeleri
   - "9 Nisan": Toplantı 3 maddeleri
   - "26 Şubat": Toplantı 1 maddeleri

3. **Bölüm Yapısı**:
   - Ana bölüm başlıkları
   - Alt bölümler
   - Tarihli maddeler

## Uygulama Detayları

### Veri Akışı
1. Toplantı notları, WhatsApp içeriklerini ve MVP bilgilerini çek
2. Tüm içeriği maddelendir ve tarih etiketle
3. İçeriği analiz edip kategorilere ayır
4. Renk kodlaması uygula
5. Veriyi Next.js/Supabase'e yükle
6. MVP bilgilerini mvpitems sayfasına "GENEL ÖZET" kartı olarak ekle
7. MVP source section'ı sil

### Veri Modeli
```typescript
interface MeetingNote {
  id: string;
  title: string;
  content: string;
  date: Date;
  color: string; // Google renk kodu
  category: string;
  subcategory?: string;
  source: 'meeting1' | 'meeting2' | 'meeting3' | 'whatsapp';
  tags: string[];
}
```

### Sayfa Bileşenleri
- `MeetingNotesHeader`: Renkli özet bölümü
- `MainSection`: Ana bölümler
- `SubSection`: Alt bölümler  
- `NoteItem`: Tarihli madde
- `CategoryCards`: Kategori kartları
- `DateBasedCards`: Tarih bazlı kartlar

### Görünüm Özellikleri
- Karanlık tema
- Renkli kategorizasyon (Google renkleri)
- Hiyerarşik bölüm yapısı
- Tarih etiketli maddeler
- Ana liste + kategori kartları + tarih kartları yapısı
- MVP items sayfası: İlk liste akordeon formatında (https://dashboard.corteqs.net/mvpitems)
- MVP genel özet: https://dashboard.corteqs.net/mvp içeriği "GENEL ÖZET" kartı olarak mvpitems sayfasına eklenip bu section silinecek, yapılandırılmış formatta gösterilecek

## Veri Çekme ve Analiz Planı

### 1. İçerik Toplama Stratejisi
- **Web Scraping Approach**: Next.js sunucu tarafında içerik çekme
- **API Integration**: Mevcut Supabase veritabanı kullanarak içerikleri yönetme
- **Content Structure Analysis**: 
  - Başlık ve alt başlıkları tespit etme
  - Madde işaretli içerikleri ayırma
  - Tarih etiketlerini otomatik tespit etme

### 2. Kategorilendirme ve Renklendirme
- **Otomatik Analiz Algoritması**:
  - TF-IDF ile anahtar kelime tespiti
  - Jaccard benzerliği ile kategori gruplama
  - Renk atama algoritması (konu bazlı)

- **Kategori Örnekleri**:
  - Rezervasyon Sistemi (kırmızı)
  - Kullanıcı Kısıtlamaları (yeşil)
  - Audit Kayıtları (mavi)
  - Veritabanı Tasarımı (sarı)
  - MVP Hedefleri (mor)
  - Reklam Modeli (turkuaz)

### 3. Veri Yapısı ve Depolama
- **Supabase Tabloları**:
  ```sql
  CREATE TABLE meeting_notes (
    id UUID PRIMARY KEY,
    content TEXT NOT NULL,
    date DATE NOT NULL,
    category TEXT NOT NULL,
    color TEXT NOT NULL,
    source TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT NOW()
  );
  ```

- **Veri Akışı**:
  1. İçerik çekme → 2. Metin analizi → 3. Kategorilendirme → 4. Supabase'e kaydetme

### 4. UI Implementasyon Detayları
- **Bileşen Mimarisi**:
  - `MeetingNotesPage`: Ana sayfa bileşeni
  - `NoteHeader`: Renkli özet bölümü
  - `NoteSection`: Ana ve alt bölümler
  - `NoteItem`: Tarihli madde bileşeni
  - `CategoryFilter`: Kategori filtreleme
  - `GeneralSummaryCard`: MVP genel özet kartı bileşeni

- **Stil Özellikleri**:
  - Tailwind CSS renk sınıfları kullanımı
  - Karanlık tema desteği
  - Hiyerarşik paddingleme
  - Tarih etiketleri için özel stiller
  - "GENEL ÖZET" kartı için özel formata sahip yapılandırılmış içerik gösterimi
