### 2.1 `src/components/layout/SidebarCategory.tsx`

**Eski hali:** Expand/collapse butonu + alt öğe listesi (items prop ile SidebarItem render)
**Yeni hali:** Doğrudan `<Link>` bileşeni — tıklayınca `/{slug}` rotasına gider

**Detaylı fark:**

KALDIRILAN prop'lar:
- `items: DocNavItem[]` — alt öğe listesi
- `activeItemId?: string` — aktif alt öğe takibi
- `isExpanded: boolean` — accordion açık/kapalı durumu
- `buttonRef?: RefObject<HTMLButtonElement>` — erişilebilirlik ref
- `onToggle: () => void` — accordion toggle callback

KALDIRILAN iç bileşenler:
- `<button>` accordion toggle (ChevronDown / ChevronRight ile)
- `<div id={panelId}>` genişleyen panel
- `<SidebarItem>` import ve render döngüsü

YENİ prop'lar (sadeleştirilmiş):
```typescript
interface SidebarCategoryProps {
  slug: string        // → Link href: /{slug}
  title: string       // → Görünen metin
  icon?: ReactNode    // → İsteğe bağlı ikon
  active?: boolean    // → Aktif sayfa vurgusu
  onItemSelect: () => void  // → Mobile sidebar kapatma
}
```

Render yapısı:
```
<Link href={`/${slug}`} onClick={onItemSelect} className={aktif ? gradient : düz}>
  {icon && <span>{icon}</span>}
  <span>{title}</span>
</Link>
```

Aktif stil: `border-[rgba(66,133,244,0.16)] bg-[linear-gradient(90deg,rgba(66,133,244,0.12),...)] text-primary-700`
Pasif stil: `border-transparent bg-white/70 text-gray-500 hover:border-[rgba(66,133,244,0.1)]`

### 2.4 `src/pages/todo.tsx` — YENİ DOSYA

Tam dosya içeriği ve açıklaması:

#### 2.4.1 Tip Tanımları
```typescript
interface TodoRow {
  id: number
  konu: string      // Görev konusu
  durum: string     // Beklemede | Devam Ediyor | Tamamlandı
  tarih: string     // YYYY-MM-DD formatında
  created_at: string
}

const DURUM_OPTIONS = ['Beklemede', 'Devam Ediyor', 'Tamamlandı'] as const

const DURUM_COLORS: Record<string, string> = {
  Beklemede:     'bg-yellow-100 text-yellow-800',
  'Devam Ediyor': 'bg-blue-100 text-blue-800',
  Tamamlandı:    'bg-green-100 text-green-800',
}
```

#### 2.4.2 State Yönetimi
```typescript
const [todos, setTodos] = useState<TodoRow[]>([])
const [konu, setKonu] = useState('')
const [durum, setDurum] = useState<string>('Beklemede')
const [tarih, setTarih] = useState(() => new Date().toISOString().slice(0, 10))
const [loading, setLoading] = useState(false)

const supabase = getSupabaseBrowserClient()
```

- `supabase` null olabilir (env yoksa) — tüm fonksiyonlar null check yapıyor

#### 2.4.3 Fonksiyonlar

| Fonksiyon | İşlem | Supabase çağrısı |
|---|---|---|
| `fetchTodos()` | Sayfa yüklendiğinde tüm görevleri çeker | `.from('todos').select('*').order('created_at', { ascending: false })` |
| `handleAdd(e)` | Form submit → yeni görev ekler | `.from('todos').insert({ konu, durum, tarih })` |
| `handleDelete(id)` | Görev siler | `.from('todos').delete().eq('id', id)` |
| `handleStatusChange(id, newDurum)` | Durum günceller | `.from('todos').update({ durum: newDurum }).eq('id', id)` |

**Akış:**
1. `useEffect` → `fetchTodos()` çağrılır
2. Kullanıcı form doldurur → `handleAdd` → Supabase insert → `fetchTodos()` ile liste yenilenir
3. Durum dropdown değişimi → `handleStatusChange` → Supabase update → local state güncellenir (re-fetch yok)
4. Sil butonu → `handleDelete` → Supabase delete → local state filterelenir (re-fetch yok)

### 2.5 `supabase/migration_002_todos.sql` — YENİ DOSYA

```sql
CREATE TABLE IF NOT EXISTS todos (
  id BIGSERIAL PRIMARY KEY,
  konu TEXT NOT NULL,
  durum TEXT NOT NULL DEFAULT 'Beklemede',
  tarih DATE NOT NULL DEFAULT CURRENT_DATE,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

ALTER TABLE todos ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Public read todos" ON todos FOR SELECT USING (true);
CREATE POLICY "Public insert todos" ON todos FOR INSERT WITH CHECK (true);
CREATE POLICY "Public update todos" ON todos FOR UPDATE USING (true) WITH CHECK (true);
CREATE POLICY "Public delete todos" ON todos FOR DELETE USING (true);
```

**Çalıştırma:** Supabase Dashboard → SQL Editor → yapıştır → Run
